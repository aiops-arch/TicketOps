        │                                                                  │

## 6. CSS — the core disease

- **23,189 lines / 653 KB**, **3,982 `!important`** (~17% of lines), **13 stacked redesign layers**.
- The **intended-final "NAVY SIDEBAR REDESIGN" is only lines 22783–23189 = 406 lines (1.75%)**. Everything else is sediment fighting it.
- **5+ full `.topbar` redefinitions** (380, 5878, 8265, 9462, 22808); **7 conflicting `#manager` grids**; **16 competing `.view.is-active` rules**; **2 duplicate mobile shells** both forcing `width:100%`.

**Bug root-causes (exact lines):**
- **Manager 0×0** → rigid grid `minmax(520px,…) minmax(420px,…)` needs ≥940px (line 22025) + the view falls through `.view{display:none}` (1370) when not active at measure time.
- **Mobile sidebar full-width** → two pre-navy mobile shells (5878 + **9462 wins**) set `.topbar{width:100%!important}`; navy mobile nav was never written.
- **Scroll clip** → Overview summary mode only: `height:100vh + overflow:hidden` (10008–10009, 9365, 10560). History/Reports are NOT clipped (false positive).

**Verdict: rewrite the CSS from the navy layer + a thin token base. Do not patch 23k lines of override wars.**

## 8. Security — header-only auth (NOT PROPER)

The auth model: login validates the password **once**, then **every** API call authenticates with only `X-TicketOps-User: <userId>` — **no token, no signature, no session secret**. Combined with `/api/auth/demo-users` returning **every user's ID with no authentication**, the identity header is trivially forgeable.

- Passwords *are* stripped from responses ✅ (`publicUser`).
- But the gate on `/api/reset`, restore, delete, and password-reset is only `requireAdmin`, which checks the **forgeable header role**. 🔴
- Apps Script deployment is `ANYONE_ANONYMOUS` 🔴 (necessary for the no-preflight design, but widens exposure).

**This is acceptable for an internal demo; it is not acceptable for production with real business data.** A real fix needs signed tokens (e.g. short-lived HMAC/JWT issued at login, verified server-side per request).

## 9. Recommended scope (if redesigning)

1. **Keep the backend logic** (`Code.gs`) — it's clean and holds live data. Add: signed-token auth + a retry/timeout wrapper note.
2. **Rewrite `styles.css`** from the 406-line navy layer + tokens. Fixes Manager, mobile, scroll, contrast in one coherent pass.
3. **Add transport resilience in `app.js`**: timeout + retry + a real "backend waking up" state (kills the 33 s silent wait + zero-data render).
4. **Optimize assets**: compress the 734 KB logo to <30 KB; tree-shake dead JS (~700 lines) and the dead CSS layers.
5. **Delete dead code**: `customer.*`, `server.js` scripts, browser-fallback mock, ghost views, unused deps.
